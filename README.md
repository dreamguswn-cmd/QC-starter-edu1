# AI Agent 품질관리·운영 모니터링 플랫폼

> SW 품질관리 및 모니터링 실무 10일 과정 — **4조 팀 프로젝트 최종 발표** (2026-07-10)

![인포그래픽](docs/인포그래픽.png)

## 프로젝트 소개

FastAPI 기반 교육과정 안내 챗봇의 **답변 품질을 자동으로 검증**하고, 기능 테스트·성능 테스트·운영 모니터링을 하나로 통합한 **QA 자동화 플랫폼**입니다.

핵심 메시지는 하나입니다 — *"기능 테스트 PASS만으로는 부족하다. 성능·모니터링·결함관리까지 검증해야 진짜 운영 가능 판정이 나온다."*

## 주요 성과

| 항목 | 결과 |
|---|---|
| 기능 테스트 (pytest) | **25/25 전체 PASS** |
| 품질 파이프라인 (rule 모드) | **7/7 PASS** |
| k6 성능 테스트 | 오류율 **0.00%** · p95 **2.3ms** (기준: 5% / 1000ms) |
| 모니터링 | Prometheus + Grafana 4개 지표 실시간 관측 |
| 결함 관리 | Jira(WT4-26~31) 등록 5건 **전건 Closed** |
| 최종 판정 | **조건부 운영 가능** (rule 모드 기준 충족, api 모드는 정책 확정 후 확대) |

## 하이라이트 — DEF-004: "서버 장애가 아니라 테스트 장애였다"

1차 k6 부하테스트에서 **141,375건 요청이 100% 실패**했습니다. 서버 결함처럼 보였지만, 분석 결과 원인은 테스트 스크립트의 `vus: 10000` 오설정(문서 기준은 `vus: 3`)이었고, 좀비 소켓이라는 2차 피해까지 발생했습니다.

1차 실행을 **테스트 무효** 처리하고 설정을 수정해 재실행한 결과, **오류율 0.00% · p95 2.3ms**로 합격 기준을 크게 상회하며 통과했습니다.

> 교훈: **테스트 도구와 실행 환경도 검증 대상이다.**

## 시스템 구성

```
사용자 ──▶ AI Agent (FastAPI, /ask rule·api 모드)
              │
              ▼
        Judge / Rule Validator (자동 품질평가)
              │
              ▼
        보고서 · Jira (결함 등록·추적)

  k6 부하 ──▶ FastAPI ◀── pytest 25건
  FastAPI /metrics ──▶ Prometheus ──▶ Grafana (실시간 대시보드)
```
