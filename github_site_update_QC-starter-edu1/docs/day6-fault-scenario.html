Day 6 · Fault Scenario Lab

장애 시나리오 실습

운영 중인 AI Agent 서비스에서 발생 가능한 응답 지연, HTTP 500, Timeout 상황을 의도적으로 재현하고 Prometheus·Grafana·로그를 통해 분석한 QA 실습입니다.

홈으로10일 과정

실습 목표

장애 재현

FastAPI 서비스에서 delay, error500, timeout 시나리오를 재현합니다.

장애 감지

Prometheus 지표와 Grafana 대시보드로 오류율과 p95 응답시간 변화를 확인합니다.

로그 분석

JSON 로그에서 요청 URL, 상태코드, 응답시간, 에러 메시지를 확인합니다.

결함보고서

재현 절차, 기대/실제 결과, 영향도, 원인 추정, 조치 방안을 정리합니다.

장애 대응 프로세스

장애 발견 → 장애 재현 → 증상 확인 → 영향도 판단 → 로그 및 지표 확인 → 결함 등록 → 조치 요청 → 재시험 → 종료/재발 방지

기대 결과 / 실제 결과기대 상태코드와 실제 상태코드, 응답시간, 메시지

포트폴리오 강조 문장

이 프로젝트의 핵심은 장애를 만드는 것이 아니라, 장애를 운영 관점에서 관리하는 능력을 보여주는 것입니다.

장애 재현 → 모니터링 → 로그 분석 → Jira 등록 → 원인 분석 → 수정 → 재시험 → 결함 종료의 전체 사이클을 경험하고 문서화했습니다.
